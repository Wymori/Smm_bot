# Автообновление CLAUDE.md

После каждого ответа пользователю проверяй, нужно ли обновить `CLAUDE.md` в корне проекта.

## Когда обновлять

Обновляй CLAUDE.md если в текущем промпте произошло что-то из списка:
- Добавлена новая модель БД или изменены существующие (поля, связи, индексы)
- Добавлен новый роутер/хэндлер или изменена структура callback data
- Изменены FSM-состояния (новые, удалённые, переименованные)
- Добавлены новые клавиатуры или изменены существующие фабрики
- Изменена архитектура (новые middleware, сервисы, утилиты)
- Подключены ранее неиспользуемые зависимости (APScheduler, Pillow, AI API)
- Изменены команды запуска или конфигурация (.env, alembic)
- Создана новая функциональность, отмеченная в секции "Что НЕ реализовано"
- Перенесена функциональность из "Что НЕ реализовано" в "Что реализовано"

## Когда НЕ обновлять

Не трогай CLAUDE.md при:
- Исправлении багов без изменения архитектуры
- Косметических правках текста/эмодзи в UI
- Рефакторинге без изменения публичного API
- Обновлении только содержимого существующих хэндлеров

## Как обновлять

1. Перечитай текущий CLAUDE.md
2. Обнови только затронутые секции, сохраняя стиль и структуру документа
3. Кратко сообщи пользователю что именно обновил в CLAUDE.md
